Issues Found & Solutions

1. Service Worker Scope Issue

Your service worker is registered at the root scope but your app is at /TCI-BUS-QR-SCANNER/.

Solution: Update the service worker registration in both HTML files:

// Change from:
navigator.serviceWorker.register('sw.js')

// To:
navigator.serviceWorker.register('/TCI-BUS-QR-SCANNER/sw.js', { scope: '/TCI-BUS-QR-SCANNER/' })

2. Missing Local Resources in Cache

Your service worker doesn't cache local files like the manifest or potential local images.

Solution: Update sw.js to include local resources:

const urlsToCache = [
  '/TCI-BUS-QR-SCANNER/',
  '/TCI-BUS-QR-SCANNER/index.html',
  '/TCI-BUS-QR-SCANNER/dashboard.html',
  '/TCI-BUS-QR-SCANNER/manifest.json',
  '/TCI-BUS-QR-SCANNER/BUS_IMAGE.jpg',
  '/TCI-BUS-QR-SCANNER/icon-192.png',
  '/TCI-BUS-QR-SCANNER/icon-512.png',
  // ... external resources
];

3. Firebase Offline Persistence

You've implemented Firestore offline persistence correctly, which is good!

How to Fix Your App

  1. Update the service worker registration in both HTML files as shown above
  2. Update the sw.js file to cache all necessary resources with the correct paths
  3. Test offline functionality again after making these changes

After making these changes, your app should work offline. Users will be able to:

  • Access the scanner and dashboard when offline
  • Scan QR codes and store results locally
  • Sync data when coming back online
